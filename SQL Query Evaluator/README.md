# SME Query Validator

A Streamlit-based application that lets a Subject Matter Expert (SME) compare a SQL query generated by a self-service tool against their own gold-standard reference query. The system uses an LLM (Claude Sonnet 4.6 via Databricks) to evaluate the queries across three weighted dimensions and produce a PASS/FAIL verdict.

---

## Overall Workflow

```
┌──────────────────────────────────────────────────────────────────┐
│                     Streamlit UI (app.py)                        │
│  User pastes: Business Question + Generated SQL + SME SQL        │
│  Clicks "Compare Queries"                                        │
└────────────────────────────┬─────────────────────────────────────┘
                             │
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│                  Orchestrator (tools.py)                          │
│  Strips SQL comments, initializes ConversationMemory,            │
│  runs all 3 tools sequentially with chained context              │
│                                                                  │
│  ┌────────────┐    output    ┌────────────┐   output   ┌──────┐ │
│  │  Tool 1    │───────────►  │  Tool 2    │──────────► │Tool 3│ │
│  │ Metric ID  │              │ SQL Equiv. │            │Result│ │
│  │  (40%)     │              │  (40%)     │            │(20%) │ │
│  └────────────┘              └────────────┘            └──────┘ │
│                                                                  │
│  Final Score = 0.40×T1 + 0.40×T2 + 0.20×T3                      │
│  PASS if ≥ 70%, else FAIL                                        │
└──────────────────────────────────────────────────────────────────┘
         │                          │                    │
         ▼                          ▼                    ▼
┌─────────────────┐  ┌──────────────────┐  ┌─────────────────────┐
│ LLM Provider    │  │ Prompt Templates │  │ SQLite Database      │
│ (providers.py)  │  │ (prompts.py)     │  │ (convert_db.py)      │
└─────────────────┘  └──────────────────┘  └─────────────────────┘
```

### Step-by-step Flow

1. **Database Setup** — On first launch, `convert_db.py` converts the CSV files under `Database/Sample Data/` into a SQLite database (`SyntheticClaims.db`).
2. **User Input** — The SME enters a business question, a generated SQL query, and their own gold-standard SQL query in the Streamlit UI.
3. **Tool 1 (Metric Identification, 40%)** — The LLM analyzes whether the generated query selects the correct KPIs, metric definitions, and dimensional grain compared to the SME query.
4. **Tool 2 (Query Formulation, 40%)** — Receives Tool 1's output as additional context. The LLM decomposes both queries into semantic components (tables, joins, filters, GROUP BY, aggregations, SELECT columns) and computes Jaccard similarity for each.
5. **Tool 3 (Result Agreement, 20%)** — Receives output from Tools 1 and 2. Both queries are executed against the SQLite database, and the LLM compares the actual result sets using relative error (scalar) or top-K overlap / correlation / value-level agreement (distributions).
6. **Final Score** — A weighted sum is computed and displayed as PASS (≥70%) or FAIL.

## File Descriptions

### `app.py`
The **Streamlit UI entry point**. Defines the page layout with:
- A text area for the business question
- Two side-by-side SQL editors (generated vs. gold standard)
- A "Compare Queries" button that triggers the evaluation pipeline
- Score summary with metrics for each tool and the final weighted result
- Three detailed tabs showing reasoning, Jaccard component tables, and executed query results

Run with: `python -m streamlit run app.py`

### `prompts.py`
Contains the **prompt templates** for all three evaluation tools. Each prompt follows the evaluation criteria from `Prompt Validator.md`:
- **`SCHEMA_CONTEXT`** — Shared database schema description and metric catalog (GCO/MIS definitions) injected into Tool 1 and Tool 2 prompts.
- **`TOOL1_METRIC_IDENTIFICATION_PROMPT`** — Chain-of-thought prompt that guides the LLM through 6 steps: identify SME metrics, identify generated metrics, compare mappings, verify grain, calculate score, and self-check.
- **`TOOL2_QUERY_FORMULATION_PROMPT`** — Mathematical prompt using Jaccard similarity across 5 equally-weighted SQL components (tables/joins, filters, GROUP BY, aggregations, SELECT). Includes prior Tool 1 context.
- **`TOOL3_RESULT_AGREEMENT_PROMPT`** — Statistical comparison prompt that handles both scalar results (relative error) and distributions (top-K overlap, Pearson correlation, value-level agreement). Includes prior Tool 1 + Tool 2 context.
- **Helper functions** (`format_tool1_prompt`, `format_tool2_prompt`, `format_tool3_prompt`) that inject variables into the templates.

### `providers.py`
Implements the **LLM provider layer**, following the `llm_gatewayV2` architecture pattern:
- **`BaseProvider`** — Abstract class defining the `chat()` interface with support for tools, reasoning, structured output, and system prompts.
- **`DatabricksClaudeProvider`** — Concrete implementation for Claude Sonnet 4.6 via Databricks serving endpoints (OpenAI-compatible API). Features:
  - Message translation (system, user, assistant, tool roles)
  - Response format handling (`json_schema` / `json_object`)
  - Reasoning effort support for extended-thinking models
  - Retry logic: falls back if `reasoning_effort` or `json_schema` is rejected by the endpoint
  - Auto-parsing of JSON responses
- **`build_provider()`** — Factory function that loads credentials from `.env` files (`DATABRICKS_HOST`, `DATABRICKS_MODEL`, `DATABRICKS_API_KEY`).

### `tools.py`
Contains the **three evaluation tool functions** and the orchestrator:
- **`run_tool1_metric_identification()`** — Sends the metric identification prompt to the LLM, parses the JSON response, and returns a `MetricIdentificationResult`.
- **`run_tool2_query_formulation()`** — Sends the SQL equivalence prompt (with Tool 1 output as context), parses Jaccard scores per component, and returns a `QueryFormulationResult`.
- **`run_tool3_result_agreement()`** — Executes both SQL queries against SQLite, short-circuits with 1.0 if results are identical, otherwise sends results to the LLM for statistical comparison. Returns a `ResultAgreementResult`.
- **`run_full_comparison()`** — Orchestrator that chains the three tools sequentially through a shared `ConversationMemory`, computes the weighted final score, and returns a `FinalScore`.
- **Helper utilities**: SQL comment stripping, JSON extraction from LLM responses, safe SQL execution (SELECT/WITH only), and result formatting.

### `schemas.py`
**Pydantic models** used across the application:
- **LLM layer**: `ChatRequest`, `ChatResponse`, `ToolDef`, `ToolCall`, `CacheableSystemBlock`, `ResponseFormat` — matching the `llm_gatewayV2` schema patterns.
- **Tool results**: `MetricIdentificationResult`, `QueryFormulationResult`, `ResultAgreementResult` — structured outputs from each evaluation tool.
- **`FinalScore`** — Combined result with individual tool scores, weights, the weighted total, PASS/FAIL status, and detailed breakdowns.

### `convert_db.py`
**Database conversion utility**. Reads four CSV files from `Database/Sample Data/` and creates a SQLite database (`Database/SyntheticClaims.db`) with tables:
| Table | Source CSV | Description |
|---|---|---|
| `claims_tbl` | `synthetic_claims_claims.csv` | One row per claim (claim-level grain) |
| `claims_events_tbl` | `synthetic_claims_events.csv` | One row per workflow event |
| `claims_lines_tbl` | `synthetic_claims_lines.csv` | One row per service line |
| `claims_flat_tbl` | `synthetic_claims_flat.csv` | Denormalized: one row per line with claim attributes |

Infers column types (TEXT vs. REAL) based on known numeric column names. Runs automatically on first app launch if the `.db` file doesn't exist.

### `Prompt Validator.md`
Evaluation rubric for assessing prompt quality. Defines 9 criteria (explicit reasoning, structured output, tool separation, conversation loop support, instructional framing, self-checks, reasoning type awareness, error handling, overall clarity) used to guide the prompt design in `prompts.py`.

### `Instructions.md`
Original project requirements and design specification.

### `requirements.txt`
Python dependencies: `streamlit`, `openai`, `pydantic`, `python-dotenv`.

## Prerequisites

- Python 3.11+
- A `.env` file with Databricks credentials:
  ```
  DATABRICKS_HOST=https://<your-workspace>.databricks.com
  DATABRICKS_MODEL=<serving-endpoint-name>
  DATABRICKS_API_KEY=<your-token>
  ```
- CSV data files present in `Database/Sample Data/`

## Quick Start

```bash
pip install -r requirements.txt
python -m streamlit run app.py
```

The app will auto-create the SQLite database on first run, then open in the browser at `http://localhost:8501`.
